### 核心玩法架構

```
白天：探索工廠
  → 翻找箱子、角落、隱藏空間
  → 發現破損玩具、零件、記憶碎片、線索
  → 每個區域有新東西，隨修復進度解鎖新區域

晚上：修復工作台
  → 用找到的零件修復玩具
  → 玩具說出故事，女主角觸發記憶
  → 決定玩具去留（送走、留下、繼續修）
  → 解鎖隔天新區域
```

## 故事與世界觀

### 世界觀
玩具世界與人類世界平行存在。玩具有意識、有情感。情感強烈的人類靈魂死後有時會投胎進玩具。這座工廠收容被退貨、遺棄、損壞的玩具，表面是修復中心，實際上是某人刻意設計來困住特定靈魂的陷阱。

### 女主角
- **結依 / Yui**，外表是普通布偶娃娃，眼睛是鈕扣
- 現在的名字是算命後改的；改名前有舊名字（待定），是伏筆核心
- 小時候暱稱「文文」——時鐘人記憶中的名字，玩家第一次聽到不會發現是主角
- 三條並行動機：找回記憶、尋找家人、發現害她死的仇人
- 仇恨很晚才出現，最初動力是愛

### 主線三條線
- **A線：** 修復玩具 → 每個玩具的故事（主要玩法）
- **B線：** 找回記憶 → 透過修復拼回前世（推進器）
- **C線：** 發現仇人 → 最終對決（高潮結局）

### 結局（開放式）
- **放下** — 留在玩具世界，繼續修復破損靈魂
- **回去** — 找到仇人，面對他，結局留白

## Godot 技術筆記

Ken 有網頁程式開發經驗，類比 CSS/JS 概念最有幫助。

### UI 定位系統（Control 節點）

**Anchor（錨點）：** 控制節點的大小和位置
- `set_anchors_and_offsets_preset(Control.PRESET_FULL_RECT)` = 撐滿父節點（≈ CSS `width:100%; height:100%`）

**Offset（偏移）：** 在 anchor 基礎上微調
- `offset_right = -12` → 右邊界往內縮 12px

**文字對齊（Label）：**
- `horizontal_alignment = HORIZONTAL_ALIGNMENT_RIGHT` (≈ CSS `text-align: right`)
- `vertical_alignment = VERTICAL_ALIGNMENT_CENTER`

**兩套座標系統：**
- UI 元素（對話框、血條）→ Control 節點 → anchor 系統
- 遊戲世界（角色、道具）→ Node2D → position 座標

**靜態 vs 動態：**
- 靜態初始值（位置、字體大小、顏色）→ 編輯器設定
- 執行時會變動的值（動畫切換、顯示/隱藏）→ 程式碼

### Timer 用法
```gdscript
get_tree().create_timer(秒數).timeout.connect(函式名, CONNECT_ONE_SHOT)
# CONNECT_ONE_SHOT = 只觸發一次就自動斷開，適合打字機逐字效果
```

### Godot 內建 Enum 常數
不需要 import，直接用。格式：`類別名.常數名`（如 `Control.PRESET_FULL_RECT`）
